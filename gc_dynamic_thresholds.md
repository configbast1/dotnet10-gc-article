# Динамічні пороги збору сміття у .NET 10

.NET 10 GC впроваджує **динамічне налаштування порогів алокації**:

- GC автоматично регулює, коли починати збір сміття залежно від:
  - навантаження на CPU,
  - швидкості алокації пам’яті,
  - кількості живих об’єктів;
- Використовуються **метрики реального часу**, які збирає runtime.

### Переваги:

- Менші паузи при пікових навантаженнях;
- Оптимізація використання пам’яті для багатопотокових програм;
- Зниження ризику OOM (OutOfMemoryException).

### Приклад конфігурації:

{
  "GCSettings": {
    "DynamicThresholds": true,
    "MaxHeapSize": "2GB"
  }
}
