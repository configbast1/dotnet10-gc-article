# Паралельна компактація у .NET 10 GC

.NET 10 використовує **паралельну компактацію купи** для зменшення часу пауз:

- Кілька сегментів купи обробляються одночасно;
- Зменшується фрагментація пам’яті;
- Покращується продуктивність великих додатків та робота з LOH (Large Object Heap).

### Вплив на реальні додатки:

- Зниження латентності для real-time систем (фінансові торги, IoT);
- Менші паузи GC при високих навантаженнях;
- Підвищення стабільності великих хмарних сервісів.
