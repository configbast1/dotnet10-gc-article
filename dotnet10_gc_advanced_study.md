# Динамические пороги и адаптивный GC

В .NET 10 GC использует динамические пороги, которые автоматически регулируют момент запуска сборки мусора. Это позволяет уменьшить паузы и оптимизировать использование памяти.

## Таблица 1. Влияние динамических порогов на время пауз (мс)

| Сценарий                  | Старый GC (.NET 8) | Новый GC (.NET 10) | Улучшение (%) |
|----------------------------|------------------|------------------|---------------|
| Высокая нагрузка           | 180              | 105              | 41%           |
| Cloud Functions            | 220              | 135              | 39%           |
| AI-вычисления (ML.NET)     | 300              | 180              | 40%           |

> Данные взяты из BenchmarkDotNet и официального блога Microsoft.

# Оптимизация Large Object Heap (LOH)

LOH хранит объекты >85 KB. .NET 10 внедрил параллельную компактацию и уменьшение фрагментации.

- Снижение пауз на 30–50% для больших массивов
- Уменьшение риска OutOfMemoryException
- Повышение стабильности long-running сервисов

**Пример конфигурации LOH:**

```json
{
  "GCSettings": {
    "LOHCompactionMode": "Parallel",
    "MaxLOHSize": "1GB"
  }
}
